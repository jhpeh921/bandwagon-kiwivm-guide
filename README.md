## KiwiVM 是什么

KiwiVM 是搬瓦工（BandwagonHost）官方自研的 VPS 控制面板，全称 KiwiVM Control Panel。搬瓦工所有套餐都跑在 KVM 虚拟化架构上，KiwiVM 就是你管理这台机器的唯一入口。

## 面板首页：先把这几个数字看清楚

打开 KiwiVM，首页信息密度不低，但真正需要关注的就几块：

**1. IP 和 SSH 端口**

这两个是连接 VPS 最基础的信息。搬瓦工的 SSH 端口不是默认的 22，每台机器独立分配，所以连不上的时候先检查这里，别一直用 22 试。

**2. 流量用量（Bandwidth usage）**

这里显示本月已用流量和重置日期。流量是按自然月计算的，超量后机器会被暂停，等下个月自动重置。要是提前用完了也可以在官网后台买额外流量包。

**3. 资源使用情况**

内存、Swap、硬盘的实时用量。Swap 显示红色的话说明内存压力比较大，通常需要优化一下跑的服务。

**4. 机器状态和操作按钮**

Start / Stop / Restart 就在这里。机器卡死的时候，先试 Restart，不行就 Stop 再 Start。

## 高频操作详解

### 重装系统

菜单找 **Install new OS**，下拉列表里选你想要的系统版本，点安装。

几个选择建议：

1. Ubuntu 24.04 LTS：目前最稳，兼容性好，大多数人选这个
2. Debian 12：轻量一些，资源占用低
3. 带 `bbr` 后缀的系统（比如 `centos-7-x86_64-bbr`）：内置 Google BBR 加速算法，能提升网络传输速度，新手可以直接选这类

注意：重装系统会清空所有数据。有重要内容先做快照。

### 重置 root 密码

菜单里找 **Root password modification**。点一下生成新密码，面板上直接显示，复制走就行。VPS 要处于运行状态才能操作，如果是已关机状态会提示失败。

### 迁移机房

**Migrate to another DC** 是 CN2 GIA-E 套餐最有价值的功能之一。买一个套餐，能在十二个以上机房之间自由切换，免费、次数不限。

哪个机房慢了？直接迁到另一个试试。迁移会保留数据，会消耗一定流量（迁移时传输数据产生），操作前最好备份一下。

迁机房会换 IPv4 地址——这点要特别注意，IP 被封的时候迁机房是个常见解决思路，但如果 IP 封得很死、无法发起迁移请求，就需要先花钱换个干净 IP。

### 快照和备份

**Snapshots** 可以手动创建快照，最多免费保存两份。重装系统或者大改配置前，先打个快照是个好习惯。

**Automatic backups** 是自动备份，每隔几天自动来一次，不需要手动操作，完全免费。

### 二次验证

**Two-factor authentication** 在 Security & Records 里，可以给 KiwiVM 登录加一层安全验证。如果对账号安全有要求，建议开启。

### Root Shell

这是个应急工具。如果 SSH 连不上了，可以在面板里直接用 Root Shell 执行命令。有三种模式：基础（不支持实时输出）、高级（日志形式返回结果）、交互式（类似 VNC，支持实时交互但不能直接复制粘贴）。紧急情况下救场用的。

## 新手常见问题

**Q：KiwiVM 和搬瓦工官网后台是同一个账号密码吗？**

不是。搬瓦工官网后台是一个账号，KiwiVM 面板有独立的登录密码，首次进入时系统会提供，也可以在 KiwiVM 里单独修改（Security 区域的 KiwiVM password modification）。

**Q：搬瓦工 KVM 面板支持中文吗？**

面板本身是英文的，没有中文界面选项，但功能不复杂，配合这篇教程对照着看，上手没什么门槛。

**Q：流量快用完了怎么办？**

在官网后台可以购买额外流量包（Bandwidth Top-up）。也可以等下个月自动重置——重置日期在 KiwiVM 首页的 Bandwidth usage 区域能看到。

**Q：IP 被封了怎么处理？**

两条路：一是在 KiwiVM 里直接花费一定费用更换 IP；二是用 Migrate 功能迁移到别的机房，机房换了 IP 自然也换了。如果当前 IP 已经完全无法连接，迁移可能也没法操作，需要先联系搬瓦工客服处理。

**Q：支持支付宝付款吗？**

支持。结账时付款方式里有支付宝选项，国内用户友好。
